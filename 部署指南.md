# 异常信息填报系统 - 超简化部署指南

## 第一步：注册GitHub（2分钟）

1. 打开 **https://github.com**
2. 点击 "Sign up" 注册账号
3. 填写用户名、邮箱、密码完成注册
4. 验证邮箱即可

> 💡 有GitHub账号直接跳过

---

## 第二步：上传代码到GitHub（3分钟）

### 方法A：网页上传（最简单）

1. 登录GitHub后，点击右上角 **"+"** → **"New repository"**
2. Repository name填写：`anomaly-report`
3. 选择 **"Public"**（免费部署需要）
4. 点击 **"Create repository"**
5. 在新页面找到 **"uploading an existing file"**
6. 把本项目所有文件拖拽到上传区域
7. 点击 **"Commit changes"**

### 方法B：Git命令上传

```bash
# 在项目文件夹打开终端
cd 异常信息填报系统
git init
git add .
git commit -m "initial commit"
git branch -M main
git remote add origin https://github.com/你的用户名/anomaly-report.git
git push -u origin main
```

---

## 第三步：注册Render并部署（5分钟）

1. 打开 **https://render.com**
2. 点击 **"Get Started"** 或 **"Sign Up"**
3. 选择 **"GitHub"** 登录（一键完成）
4. 点击 **"New +"** → **"Web Service"**
5. 在 "Connect a repository" 页面找到你的 `anomaly-report` 仓库
6. 点击连接
7. 填写配置：
   - **Name**: `anomaly-report`（或你喜欢的名字）
   - **Region**: Singapore（延迟最低）
   - **Branch**: `main`
   - **Root Directory**: （留空）
   - **Runtime**: `Python 3`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `gunicorn app:app`
8. 点击 **"Create Web Service"**
9. **等待构建完成**（约1-2分钟）

---

## 第四步：配置数据库（2分钟）

1. 在Render dashboard点击 **"New +"** → **"PostgreSQL"**
2. Name填写：`anomaly-db`
3. 选择 **"Free"** 计划
4. 点击 **"Create Database"**
5. 等待创建完成（约30秒）
6. 复制 **"Internal Database URL"**
7. 回到你的 Web Service 页面
8. 点击 **"Environment"** → **"Add Environment Variable"**
9. Key填写：`DATABASE_URL`
10. Value粘贴刚才复制的数据库地址
11. 点击 **"Save Changes"**
12. 系统会自动重启

---

## 第五步：完成！获取访问地址

1. 回到 Web Service 页面
2. 在顶部找到你的应用地址，如：
   ```
   https://anomaly-report.onrender.com
   ```
3. 点击链接即可访问系统！

## 常见问题

### Q: 构建失败怎么办？
- 检查Build Command是否正确
- 查看Build Logs定位错误

### Q: 页面显示500错误？
- 检查DATABASE_URL是否配置
- 查看Application Logs

### Q: 数据会丢失吗？
- Free计划数据库在30天后无连接会休眠
- 建议定期访问保持活跃

### Q: 如何更新代码？
- 更新GitHub代码后，Render会自动重新部署
